# 修正 CAS 重定向 401 响应处理

## 摘要

- 保留 `Dio` 全局 `validateStatus: (status) => status != null && status < 400` 判断不变。
- 修正 CAS 重定向拦截器内部请求遇到 `401` 时直接抛出 `DioException [bad response]` 的问题。
- 当重定向内部请求收到带 `response` 的 `DioException` 时，通过 `handler.resolve(e.response!)` 把响应交回调用方处理，避免在拦截器中中断登录流程。
- 执行 `dart format lib\dio_http.dart`。
- 执行 `flutter analyze`，当前仍有既有 `avoid_print` 信息和 `lib/home.dart` 未使用成员警告。
